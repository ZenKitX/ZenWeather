# 快速参考

## 🚀 常用命令

### 开发命令

```bash
# 获取依赖
flutter pub get

# 运行应用
flutter run

# 热重载
r

# 热重启
R

# 清理构建
flutter clean
```

### 代码质量

```bash
# 格式化代码
dart format .

# 检查格式（CI 使用）
dart format --set-exit-if-changed .

# 代码分析
flutter analyze

# 运行测试
flutter test

# 生成覆盖率
flutter test --coverage
```

### 构建命令

```bash
# Android APK
flutter build apk --release

# Android App Bundle
flutter build appbundle --release

# iOS (需要 macOS)
flutter build ios --release --no-codesign

# Web
flutter build web --release
```

## 📝 Git 工作流

### 日常开发

```bash
# 1. 创建功能分支
git checkout -b feature/new-feature

# 2. 开发并提交（遵循 Conventional Commits）
git add .
git commit -m "feat: 添加新功能"

# 3. 推送分支
git push origin feature/new-feature

# 4. 在 GitHub 创建 Pull Request
```

### 提交规范

```bash
# 新功能
git commit -m "feat(scope): 描述"

# Bug 修复
git commit -m "fix(scope): 描述"

# 文档更新
git commit -m "docs: 描述"

# 代码格式
git commit -m "style: 描述"

# 代码重构
git commit -m "refactor(scope): 描述"

# 性能优化
git commit -m "perf(scope): 描述"

# 测试相关
git commit -m "test: 描述"

# 构建/工具
git commit -m "chore: 描述"

# CI/CD
git commit -m "ci: 描述"

# 资源文件
git commit -m "assets: 描述"
```

### 发布流程

```bash
# 1. 更新版本号（pubspec.yaml）
version: 1.0.0+1

# 2. 更新 CHANGELOG.md

# 3. 提交更改
git add pubspec.yaml CHANGELOG.md
git commit -m "chore: 准备发布 v1.0.0"
git push origin main

# 4. 创建标签
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0

# 5. GitHub Actions 自动发布
```

## 🔧 故障排除

```bash
# 清理并重新获取依赖
flutter clean
flutter pub get

# 重置 Flutter
flutter doctor
flutter clean
rm -rf build/
flutter pub get

# 清除 Dart 缓存
dart pub cache clean

# 修复 iOS Pods（macOS）
cd ios
pod deintegrate
pod install
cd ..
```

## 📱 平台特定

### Android

```bash
# 查看签名信息
keytool -list -v -keystore android/app/keystore.jks

# 生成签名密钥
keytool -genkey -v -keystore android/app/keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias key

# 构建 APK（调试）
flutter build apk --debug

# 构建 APK（发布）
flutter build apk --release

# 安装到设备
flutter install
```

### iOS (macOS)

```bash
# 打开 Xcode
open ios/Runner.xcworkspace

# 清理 Pods
cd ios
rm -rf Pods Podfile.lock
pod install
cd ..

# 构建 iOS
flutter build ios --release

# 运行在模拟器
flutter run -d ios
```

### Web

```bash
# 运行 Web 版本
flutter run -d chrome

# 构建 Web
flutter build web --release

# 指定 base href
flutter build web --base-href /zen-weather/

# 本地预览（需要 http-server）
cd build/web
npx http-server
```

## 💡 提示

### 提高开发效率

1. 使用 `r` 进行热重载，而不是重启应用
2. 使用 `dart format .` 保持代码格式一致
3. 定期运行 `flutter analyze` 检查代码质量
4. 提交前运行 `flutter test` 确保测试通过
5. 使用有意义的提交信息

### 常见问题

**Q: 依赖冲突怎么办？**
```bash
flutter clean
flutter pub get
```

**Q: 构建失败怎么办？**
```bash
flutter clean
rm -rf build/
flutter pub get
flutter build apk
```

**Q: iOS 构建失败？**
```bash
cd ios
pod deintegrate
pod install
cd ..
flutter clean
flutter build ios
```

**Q: 如何查看详细错误？**
```bash
flutter run -v
flutter build apk -v
```
