### 2. 数据流程
```
用户位置
  ↓
LocationService.getCurrentLocation()
  ↓
WeatherService.getWeatherByLocation(lat, lon)
  ↓
WeatherAPI 返回数据
  ↓
WeatherModel 解析数据
  ↓
HomeController 更新状态
  ↓
HomeView 显示数据
```

## 📱 权限处理

### Android
```xml
<!-- 定位权限 -->
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />

<!-- 网络权限 -->
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
```

### iOS（需要配置）
在 `ios/Runner/Info.plist` 中添加：
```xml
<key>NSLocationWhenInUseUsageDescription</key>
<string>需要获取您的位置以提供当地天气信息</string>
<key>NSLocationAlwaysUsageDescription</key>
<string>需要获取您的位置以提供当地天气信息</string>
```

## 🔧 配置步骤

### 1. 获取 API Key
1. 访问 https://www.weatherapi.com/
2. 注册账号
3. 获取免费 API Key

### 2. 配置 API Key
编辑 `lib/utils/constants.dart`：
```dart
static const String weatherApiKey = '你的API密钥';
```

### 3. 运行项目
```bash
cd zen_weather
flutter run
```

### 4. 授予权限
首次运行时，应用会请求定位权限，点击"允许"。

## ⚠️ 注意事项

### 1. API Key 安全
- ❌ 不要将 API Key 提交到公共仓库
- ✅ 使用环境变量或配置文件
- ✅ 在 .gitignore 中忽略配置文件

### 2. 定位权限
- 首次运行需要授予定位权限
- 如果拒绝权限，会使用默认城市（北京）
- 可以在设置中重新授予权限

### 3. 网络连接
- 需要网络连接才能获取天气数据
- 无网络时会显示错误提示
- 可以下拉刷新重试

### 4. API 限制
- 免费版每月 100 万次调用
- 建议添加缓存机制减少调用
- 避免频繁刷新
